# AspectMind: Comprehensive Technical Documentation

## 1. Executive Summary
**AspectMind** is a state-of-the-art intelligent product recommendation architecture that bridges classical semantic vector search with advanced **Aspect-Based Sentiment Analysis (ABSA)**. The system is deliberately engineered to provide highly relevant product discoverability based on deep Neural Language Processing (NLP) models. This architecture permits users to organically search for products utilizing natural language, and critically filter them relying on granular sentiment scores extracted dynamically from past reviews.

The comprehensive design consists of a robust and offline-capable FastAPI Python backend driven by PyTorch and Transformers, which handles all computationally heavy Machine Learning tensor operations, coupled with a dynamic React 19 Frontend utilizing Vite that visualizes metrics, charts, comparisons, and seamless aspect-filtering matrices.

---

## 2. Global Directory & Module Architecture

AspectMind operates under a macro-monorepo setup comprising distinct service boundaries:

- `client/`: The React 19 frontend UI application. Contains structured Vite build tooling, dependency maps (`package.json`), and segmented UI component trees mapped under `src/components/`.
- `server/`: The primary FastAPI orchestration application. Contains the AI pipeline orchestrator (`recommender.py`), HTTP bindings (`main.py`), offline model loading setup (`setup_offline.py`), and localized python dependencies.
- `data/`: The isolated storage module housing the massive datasets (`Second_fixed_image_urls.csv`), database backups, and Pandas serialized matrix `.pkl` cache derivatives.
- `embeddings/`: Contains mathematically computed high-dimension vector search models, particularly the `faiss_index.bin`, `hnsw_index.bin`, `knn_model.pkl`, and `enriched_item_descriptions_embeddings.npy`.
- `models/`: The centralized local offline HuggingFace model cache containing:
  - `all-MiniLM-L6-v2` (for semantic search vector representations).
  - `deberta-v3-base-absa` (for aspect-based sentiment evaluations).
  - `ms-marco-MiniLM-L-6-v2` (for deep context-accuracy cross-encoding and re-ranking).
- `scripts/`: A dedicated suite of utility scripts (`diagnose_images.py`, `repair_images_fixed.py`, `fix_ames_product.py`, etc.) designed specifically to sanitize, scrape, repair CDN failures, and clean massive dataframe CSV structures before system initialization.
- `/`: The root directory contains initializing scripts such as `start_app.bat` to dual-boot the system environments.

## 4. In-Depth Dependency & Library Analysis

To accurately comprehend AspectMind's execution matrix, we must analyze its highly specialized toolchain orchestration.

### 4.1 Backend Data & ML Pipeline (`server/requirements.txt`)

**Web API Sub-System:**
- **`fastapi`**: The asynchronous web architecture utilized to rapidly construct the RESTful API routes (`/search`, `/analyze`, `/compare`). Responses utilize declarative Pydantic schemas for data mutation and type validation.
- **`uvicorn`**: The designated ASGI (Asynchronous Server Gateway Interface) server wrapping the FastAPI application, providing the asynchronous async/await event loops mandatory to handle concurrent querying.

**Data & Mathematical Array Tooling:**
- **`pandas`**: The backbone structural dependency for ingesting and manipulating the foundational dataset representations (`Second_fixed_image_urls.csv`). Responsible for rapid vectorized array transformations prior to ML processing.
- **`numpy`**: The raw computational driver for multidimensional list manipulations, matrix reshapes, and managing precision embedding outputs natively generated by language models.
- **`scipy`**: The underlying mathematical scientific operation toolkit utilized explicitly for algorithms like adjusting `sigmoid()` probabilities to compress logit boundaries uniformly between `0` and `1`.

**Deep Learning & Core NLP Foundation:**
- **`torch` (PyTorch)**: The deep learning CUDA/CPU tensor library powering all NLP operations. It administers all continuous dynamic computational graphs, weight calculations, and allocations required to traverse language models.
- **`transformers` (Hugging Face)**: Utilized dynamically via a mapped pipeline sequence to invoke the `deberta-v3-base-absa` model structure. Designed solely to execute Aspect-Based Sentiment Analysis on tokenized string boundaries.
- **`sentence-transformers`**: Generates contextually aware representation embeddings. Deploys the `all-MiniLM-L6-v2` sequence to formulate vectorized shapes of description data and `ms-marco` cross-encoder for verifying query exact-match correlation.
- **`spacy`**: An industrial-grade linguistic text tokenizer and POS (Part-of-Speech) tagger. It is rigorously mapped to extract grammatical noun subjects—the specific "aspects" or physical features isolated by natural language (e.g., extracting "battery" or "camera" from user search inputs).
- **`scikit-learn`**: Utilized for feature manipulations, data scale normalizations (like `MinMaxScaler`), and dimensionality reductions inside preprocessing hooks.

**Vector Mapping Core:**
- **`faiss-cpu` (Facebook AI Similarity Search)**: The highly optimized, C++-backed mathematical nearest-neighbors lookup engine. Effectively catalogs and sweeps `130,000+` item arrays, resolving $O(\log N)$ complex vector proximity correlations.

**Model Check-pointing & Acceleration Constraints:**
- **`joblib`**: A serialization methodology. Selected explicitly for writing/reading `.pkl` massive Pandas memory pools efficiently back into RAM.
- **`accelerate` & `sentencepiece` & `protobuf`**: Under-the-hood C++ serialization and automated model partition loaders required specifically for newer advanced transformer iterations (such as `deberta-v3-base`).
- **`tqdm`**: Command-line diagnostic visualizations used continuously to iterate massive indexing batch matrices cleanly with percentage mapping.

### 4.2 Frontend Presentation Layer (`client/package.json`)

- **`react` & `react-dom` (v19)**: The foundational HTML structural mounting engine orchestrating virtual DOM reconciliation.
- **`vite` (via Rollup ESM)**: The sub-second HMR delivery bundler that efficiently compiles, minimizes, and serves the UI dependencies statically over to Node.
- **`axios`**: The Promise-oriented dispatch controller designed specifically for routing serialized JSON payloads across REST boundaries to FastAPI (e.g., `axios.post()`).
- **`framer-motion`**: The advanced DOM physics animation module powering spring calculations, scaling animations, seamless DOM unmounting transitions without relying mechanically on CSS keyframes.
- **`react-tilt`**: A UI modifier component yielding contextual 3D spatial transformations tracking physical cursors mapped intimately over elements.
- **`recharts`**: A fully decoupled D3/React graph abstraction mechanism building the aggregate pie charts and visual diagnostics internally.
- **`lucide-react`**: Vectorized geometric standard SVG icon integration.

## 5. Algorithmic System Execution Paths

### 5.1 Backend Initialization & Global Pipeline Architectonics
Located primarily inside `server/recommender.py` and orchestrated directly through `server/main.py` bindings.

1. **System Startup Isolation**:
   The engine utilizes aggressive `TRANSFORMERS_OFFLINE` triggers directly bounding model checkpoints to the local `/models` directory to ensure flawless localized execution independent of Hugging Face availability constraints. `setup_offline.py` is capable of pre-staging these binaries.
2. **Data Framing & Cashing Methodology (`_load_data_cache`)**:
   `joblib` validates the local `data/` cache presence of serialized mappings (`Second_fixed_image_urls.csv_processed.pkl`). If uncached, the system iterates standard preprocessing constraints encapsulating data drops and text sanitation.
3. **The Index Memory Mapper (`_prepare_embeddings_and_index`)**:
   FAISS and Numpy mapping indexes are dynamically swept into active system RAM, allowing real-time geometric angular calculations on item vectors.
4. **Execution: The `recommend()` Method**:
   - **Vector Spatial Mapping**: Converts incoming semantic text to spatial values.
   - **K-NN Lookups**: FAISS executes mathematical proximities.
   - **Context Verification**: The result pool is validated against exact queries using the `ms-marco` cross-encoder.
   - **Grammar Isolation**: `spacy` infers user sentence nouns evaluating explicit targets (e.g., "fast speed", isolating "speed").
   - **Targeted ABSA Scoring**: Generates floating calculations (`sigmoid()` bounds) determining the explicitly targeted positivity logic against the found aspects inside physical product reviews using the `deberta` transformer.
   - **Algorithmic Filter Exclusions**: Rejects JSON nodes dynamically scoring beneath designated interface range requirements (`min_sentiment_score`).

- **`ProductModal.jsx` & `ProductModal.css`**: The `z-index` overlay container permitting user feedback submission bindings against backend `add_feedback()`.

## 7. Operational Workflows & Edge Scripts

### Workflow Initialization: Single-Boot Topology
Running the fundamental application via the windows `start_app.bat` executable coordinates dual-parallel process structures:
1. `cmd` spawns context over `server/`, validates missing dependency links dynamically inside `requirements.txt`, forces local environment models, and serves `main.py`.
2. `cmd` consecutively spawns context over `client/`, resolving missing static `node_modules` sub-dependencies via explicit `npm install axios/react`, surfacing the Vite host application locally accessible at `http://localhost:5173`. 

### Specialized Data Management Tooling
The `scripts/` directory is an independent execution domain necessary exclusively to correct and sanitize external database malformations continuously:
- **`diagnose_images.py` / `quick_check.py`**: Reads `.csv` formats evaluating link accessibility patterns.
- **`repair_images_fixed.py` / `fix_ames_product.py`**: Actively modifies structured CDNs, rectifying generic un-renderable string mappings, writing iteratively back to backup CSV formats stored explicitly underneath the `data/` folder hierarchy. 

**Resulting System Capability**: A perfectly aligned local ecosystem encapsulating rapid vector index lookup, decoupled ML sentiment mappings, zero-latency React presentation, and uninhibited localized model serving capabilities ensuring offline functional completeness perfectly encapsulating a single unified pipeline solution.
